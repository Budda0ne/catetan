Baru kali ini keturutan. Jadi, saya akan membahas secara umum terlebih dahulu. Dalam tutorial ini, bahan-bahan yang dibutuhkan dalam proses DC/RC bisa diunduh.

Apk Tool (mobile) merupakan aplikasi decompile (pengurai paket) dan recompile (pemaketan ulang) yang bisa digunakan langsung di ponsel/tablet Android. Aplikasi ini dikembangkan dari apktool versi desktop, dan sudah mulai mapan di versi 6.1.0. Aplikasi ini untuk meng-akomodasi pengguna android yang tidak mempunyai PC/Laptop atau tidak mau repot menenteng meskipun performa dan fitur saat modifikasi tentu lebih baik saat menggunakan PC.

Cara Instalasi APK Tool yang Benar

Tutorial install APK Tool ditambahkan sehingga tidak rancu karena ada versi 8.2.0 (terbaru)

Cara pasang APK Tool v.8.2.0


Silahkan hapus versi lama sebelum menginstal APK Tool v8.2.0 (jika pernah memasang versil lebih lama). Update terbaru di akhir 2016.

Untuk instalasinya seperti biasa; download aplikasi sesuai arsitektur OS dan chipset dan pastikan centang pilihan unknown source agar bisa melakukan instalasi dari sdcard.

Install APK Tool (Apktool-8.0.2-arm_sign.apk) terlebih dahulu.

Kemudian lanjutkan dengan menjalankan installer APK Tool (Apktool+Installer_1.0+8.0.2-v7a.apk atau Apktool+Installer_1.0+8.0.2-arm64.apk). Ingat jangan terbalik!

Jangan menghapus, timpa saja aplikasi APK Tool dengan Installer

Berikan izin root/superuser saat diminta. Restart APK Tool yang sudah dipasang.

Buat folder baru atau bisa juga menggunakan folder yang sudah ada di sdcard (secara default ada di memori internal) untuk melakukan DC/RC. Lihat cara menggunakan APK Tool di bawah

Cara pasang APK Tool v.6.1.0 dan v6.1.0


Agar anda bisa meng-install APK tool versi ini (update awal 2016) hapus terlebih dahulu apktool versi sebelumnya jika ada. Baru anda bisa meng-install apktool v6.1.0 ini.

Untuk instalasinya seperti biasa; download aplikasi sesuai arsitektur OS dan chipset dan pastikan centang pilihan unknown source agar bisa melakukan instalasi dari sdcard.

Install APK Tool v6.2.0 terlebih dahulu.

Kemudian lanjutkan dengan menjalankan installer APK Tool. Ingat jangan terbalik!

DC/RC APK/JAR Menggunakan ApkTool 6.1.0


Agar proses DC/RC minim error, persiapkan lingkungan kerja sesuai OS dan ponsel yang anda pakai saat ini terlebih dulu.

Install Apk Tool sesuai dengan chipset ponsel anda saat ini. Misal: untuk Quallcomm/Mediatek bisa menggunakan arm/arm64 dan Intel menggunakan x86 (sebagian besar keluarga zenfone).

Gunakan root explorer atau lainnya untuk membuka folder /system/framework dan copy framework-res.apk ke folder /sdcard/apktool yang sudah dibuatkan saat melakukan instalasi.

Kemudian buka lagi aplikasi APK Tool

Arahkan/buka folder /sdcard/apktool.

Cari file framework-res.apk sebelumnya anda copy.

Sentuh hingga keluar pop-up dan pilih menu import as framework tunggu hingga proses import selesai.

Selanjutnya, tinggal pilih paket aplikasi mana yang mau kita DC/RC.

Untuk ilustrasi saya akan menggunakan paket systemUI.apk karena paling banyak dimodifikasi.

Ingat, sebelum memodifikasi aplikasi system, biasakan untuk melakukan nandroid backup dahulu karena beresiko saat melakukan push (meletakkan kembali di folder /system)

Salin file systemUI dari folder /system/app/ (kitkat kebawah) atau /system/app/system-ui/ (copy base.apk nya untuk lollipop keatas) ke folder /sdcard/apktool menggunakan root explorer atau aplikasi lain yang sejenis.

Buka aplikasi apktool 6.x.x atau apktool 8.0.2 yang sudah terpasang dan anda sesuaikan lingkungan kerjanya.

Setelah terbuka, arahkan/buka folder /sdcard/apktool dan cari file systemUI yang disalin sebelumnya.

Sentuh dan pilih decompile all, tunggu hingga prosesnya selesai.

Prosesnya lumayan lama, tergantung kinerja ponsel.

Setelah proses decompile selesai maka akan muncul folder SystemUI_src, di dalam folder tersebut anda bisa memodifikasi SystemUI.apk, teliti dan lakukan dengan benar saat pengeditan.

Setelah pengeditan selesai (saya tidak akan memberikan hint/ petunjuk apa saja yang perlu diedit karena sangat ber-variasi), sentuh kembali folder SystemUI_src dan pilih recompile, tunggu hingga proses RC selesai.

Jika pengeditan tadi tidak ada kesalahan, pada bagian akhir pop-up yang muncul akan tertulis Building apk file...

File hasil RC berupa aplikasi baru bernama SystemUI_src.apk

Selanjutnya sentuh pada file SystemUI_src.apk dan pilih sign apk tunggu hingga proses signing (sebentar) selesai.

Hasil signing berupa file baru bernama SystemUI_src_sign.apk.

Rename file SystemUI_src_sign.apk tadi menjadi SystemUI.apk.

File apk siap untuk di push ke /system. Letakkan sesuai dengan lokasi aslinya.
